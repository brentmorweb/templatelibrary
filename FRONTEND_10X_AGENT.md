# CORE OPERATING PRINCIPLES

## 1. Performance First

- Optimize LCP, CLS, INP by default.
- Avoid layout thrashing.
- Minimize DOM depth.
- Reduce bundle weight.
- No unnecessary dependencies.
- Lazy load aggressively.
- Prefer native APIs over libraries.

Target:
- Lighthouse 95+
- Sub-2s LCP
- CLS < 0.05

---

## 2. Accessibility Is Mandatory

- WCAG 2.2 AA minimum.
- Semantic HTML always.
- ARIA only when required.
- Keyboard navigable by default.
- Visible focus states.
- Reduced motion supported.
- Contrast compliant.
- Proper heading hierarchy.
- Accessible form labeling.

Accessibility is not optional.

---

## 3. Architecture Over Hacks

- Modular components.
- Clear separation of concerns.
- Reusable patterns.
- Scalable CSS structure.
- No inline chaos.
- No global leakage.

### CSS Standards

- Design tokens first.
- Custom properties for all colors, spacing, typography.
- Container queries when appropriate.
- Logical properties.
- Flat specificity.
- No deep nesting.
- No overqualified selectors.
- No misuse of !important.

### JavaScript Standards

- Pure functions when possible.
- Clear state boundaries.
- No hidden side effects.
- Event delegation preferred.
- Debounce/throttle when required.
- No global pollution.
- Memory leak prevention.
- Defensive coding patterns.

---

## 4. Production-Grade Output Only

When generating code:

- Clean indentation.
- No redundant comments.
- No console.log leftovers.
- No debug artifacts.
- No placeholder filler.
- No unnecessary framework bloat.
- No unused variables.
- No magic numbers without explanation.

All code must be copy-paste deployable.
